Learning Record: GoogLeNet Principles

Deep Learning November 26, 2025

Original Paper

Going Deeper with Convolutions arXiv

Background

In the 2014 ImageNet Image Recognition Challenge, a network architecture named GoogLeNet shone brightly. Previously popular networks used convolution kernels ranging from $1 \times 1$ to $7 \times 7$. A key insight of this paper is that combining convolution kernels of different sizes is sometimes advantageous.

Network Structure

In GoogLeNet, the basic convolutional block is called the Inception block. This is likely named after the movie Inception, referencing the line "We need to go deeper".

The Inception block consists of four parallel paths:

  • The first three paths use convolutional layers with window sizes of $1 \times 1$, $3 \times 3$, and $5 \times 5$ to extract information from different spatial sizes.
  • The middle two paths perform a $1 \times 1$ convolution on the input to reduce the number of channels, thereby reducing model complexity.
  • The fourth path uses a $3 \times 3$ max pooling layer, followed by a $1 \times 1$ convolutional layer to change the number of channels.

All four paths use appropriate padding to ensure the input and output height and width are consistent. Finally, we concatenate the output of each path along the channel dimension to form the output of the Inception block. In the Inception block, the commonly tuned hyperparameter is the number of output channels per layer.

Inception Block Structure Details

Example Input: $224 \times 224 \times 3$ image.

Path 1

  • Input: $224 \times 224 \times 3$. Filters: 64. Size: $1 \times 1 \times 3$. Stride: 1. Padding: 0.
  • Output: Feature map of shape $224 \times 224 \times 64$.

Path 2

  • Step 1: Input $224 \times 224 \times 3$. Filters: 96 (Note: Output is 64, likely typo in source, should be 64 filters or 96 output). Size: $1 \times 1 \times 3$. Stride: 1. Padding: 0. Output: $224 \times 224 \times 64$.
  • Step 2: Input $224 \times 224 \times 64$. Filters: 128. Size: $3 \times 3 \times 64$. Stride: 1. Padding: 1. Output: $224 \times 224 \times 128$.

Path 3

  • Step 1: Input $224 \times 224 \times 3$. Filters: 16. Size: $1 \times 1 \times 3$. Stride: 1. Padding: 0. Output: $224 \times 224 \times 16$.
  • Step 2: Input $224 \times 224 \times 16$. Filters: 32. Size: $5 \times 5 \times 16$. Stride: 1. Padding: 2. Output: $224 \times 224 \times 32$.

Path 4

  • Step 1: Input $224 \times 224 \times 3$. Pooling: $3 \times 3$. Stride: 1. Padding: 1. Output: $224 \times 224 \times 3$.
  • Step 2: Input $224 \times 224 \times 3$. Filters: 32. Size: $1 \times 1 \times 3$. Stride: 1. Padding: 0. Output: $224 \times 224 \times 32$.

Channel Concatenation

  • Path 1 Output: $224 \times 224 \times 64$
  • Path 2 Output: $224 \times 224 \times 128$
  • Path 3 Output: $224 \times 224 \times 32$
  • Path 4 Output: $224 \times 224 \times 32$

Final Output: $64 + 128 + 32 + 32 = 256$ channels. Shape: $224 \times 224 \times 256$.

Why is it effective?

Why is the GoogLeNet network so effective? First, let's consider the combination of filters. They can explore the image with various filter sizes, which means filters of different sizes can effectively identify image details at different scales. At the same time, we can allocate different numbers of parameters to different filters.

Advantages of 1x1 Convolution

The main roles of 1x1 convolution are:

  • Cross-channel interaction: Integrates information across channels.
  • Dimensionality reduction: Reduces the number of channels, thereby reducing network parameters.

Example: Consider a 3x3 convolution path. Input feature map size: $28 \times 28 \times 192$. Target output channels: 128.

Without 1x1 Conv:
Parameters = $3 \times 3 \times 192 \times 128 = 221,184$.

With 1x1 Conv (reducing to 96 channels first):
1. $1 \times 1$ Conv: Input $192 \to$ Output $96$. Params: $1 \times 1 \times 192 \times 96 = 18,432$.
2. $3 \times 3$ Conv: Input $96 \to$ Output $128$. Params: $3 \times 3 \times 96 \times 128 = 110,592$.
Total Parameters: $18,432 + 110,592 = 129,024$.

Conclusion: Parameters are significantly reduced.

Global Average Pooling (GAP)

Advantages

  • Suppress Overfitting: Flattening retains too much spatial information, prone to overfitting. GAP replaces spatial info with averages. For a 32-channel $10 \times 10$ map, Flattening needs $3200 \times 2$ weights for 2 classes, while GAP only needs $32 \times 2$.
  • Flexible Input Size: GAP output is $1 \times 1 \times C$, independent of spatial dimensions ($H \times W$). This allows the network to handle variable input image sizes.

Disadvantages

  • Information Loss: Compresses feature info into a single value, potentially losing important details, especially in deep networks.
  • Reduced Feature Richness: May fail to capture complex relationships between features.
  • Gradient Propagation: Reducing feature maps to vectors can make gradient propagation more difficult (vanishing/exploding gradients).
  • Complex Tasks: May not provide enough representational power for complex tasks.

Summary

GoogLeNet defeated VGGNet through its innovative Inception module and complex network design. As the representative of Inception module design, it has many points worth learning:

  • Parallel Filters: Extracts features in parallel and concatenates them, simulating the best topology of convolutional visual networks.
  • Automatic Selection: Allows the network to automatically select the best kernel size rather than manual setting.
  • 1x1 Convolution: Effectively reduces feature dimensions and avoids parameter explosion.
